Interface IRandomWriter

Synchronous interface for writing randomly to a data sink.

Namespace: KdSoft.StreamUtils
Assembly: KdSoft.StreamUtils.dll
Syntax
public interface IRandomWriter : IWriter
Remarks

Intended to completely fill the underlying data entity without gaps. The implementaion may, but does not have to, ignore parts of a write operation that would overlap with already written data.

Methods

EndWrite(Byte[], Int32, Int32, Int64)

Writes end buffer to the underlying data sink, thus determining the final size of the target data stream.

bool EndWrite(byte[] buffer, int start, int count, long targetOffset)
Parameters
Type Name Description
System.Byte[] buffer

Byte buffer to write the data from.
System.Int32 start

Position in buffer to start reading from.
System.Int32 count

Number of bytes to write.
System.Int64 targetOffset

Position in underlying entity to start writing to.
Returns
Type Description
System.Boolean

true if call was successful, false if already in a completed state.
Remarks
  • This can be called at any time, that is, even before other writes have been performed.
  • If the data stream already extends beyond the end position (targetOffset + count) then the implementation may, but does not have to, throw an exception. If it does not throw, then the data stream will be truncated at the end position.
  • If the completed state was reached by calling FinalWrite(Byte[], Int32, Int32) on the same underlying entity, then this may do nothing - and return false - even on the first call.

Type Description
System.Boolean

Remarks
  • Writing beyond the end boundaries of the writable range may result in error, this is left to the implementation. The end of the range is only known after EndWrite(Byte[], Int32, Int32, Int64) was called.
  • The implementation may decide to consider the data complete once all data in the range was written at least once, and then do nothing for subsequent write requests.
