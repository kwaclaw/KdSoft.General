Class ExhaustiveReadWritePump

Pumps data from reader to writer until read count is less than requested. This operation be repeated until the end of data is reached.

Parameters
Type Name Description
System.Threading.Tasks.TaskCreationOptions options

Task creation options for returned task.
System.Boolean waitForWrites

If false the result task will complete when all reads have finished, if true the result task will additionally wait until all writes have finished.
Returns
Type Description
System.Threading.Tasks.Task<System.Boolean>

Task instance that can be waited on, or null if already in a completed state. The task result indicates if the more data are available (true) or if the end of data was reached (false).
Remarks
  • This is useful for reading from a buffer that could be temporarily exhausted.
  • The behaviour might be unpredictable if pump tasks are executing concurrently, it is best to wait for the previous PumpData task's completion before calling PumpData again.
