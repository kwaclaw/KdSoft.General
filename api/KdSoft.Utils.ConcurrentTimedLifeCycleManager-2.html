Class ConcurrentTimedLifeCycleManager<K, O>

Manages and tracks the life-cycle of objects implementing ILifeCycleAware<T>. When an object's life-cycle has expired and no other objects have a reference to it then it will be removed from the internal object map and consequently will be available for garbage collection. On the other hand, an object will be kept un-collected as long as its life-cycle has not expired, which is useful for stateful server objects.
